# Cómo funciona gvNIX te permite construir el modelo de datos de una aplicación mediante ingeniería directa e inversa y generar una aplicación de gestión (web o no) en múltiples tecnologías de visualización (Spring MVC, Flex, JSF, Vaddin..) gvNIX ha sido implementado con lenguaje Java y sigue un modelo de arquitectura orientado a componentes sobre plataforma **OSGi** en el que cada funcionalidad que ofrece el framework es implementada como un componente o add-on que colabora con el resto en las distintas tareas de generación. Está compuesto de un conjunto de herramientas de código abierto entra las que destaca su núcleo, Spring Roo. La tecnología que incluye gvNIX es la tecnología más utilizada en la actualidad con un 40% de la población de estudio emplea Spring MVC como framework. # Qué aporta a tu proyecto ## Intérprete de comandos Desde el punto de vista de su uso, gvNIX está diseñado como un intérprete de comandos interactivo al estilo de Rails o de Grails. Para facilitar su uso tiene autocompletado de los comandos y ayuda contextual. ## dandelion datatables ## Interfaz usuario adaptativa (responsive UI) con [Bootstrap](http://getbootstrap.com/) ## Internacionalización ## Componentes avanzados: tablas AJAX, lupa ## Informes # Funcionalidades TBC * Maestro → detalle multinivel # Características TBC * gvNIX puede generar cualquiera de las capas de la arquitectura de la aplicación: sólo modelo, modelo-controlador o modelo-vista-controlador. De esta forma gvNIX ayuda en aquello que necesita el proyecto y permite integrar otras tecnologías en una aplicación desarrollada con gvNIX. Por ejemplo, se puede desarrollar una aplicación con gvNIX cuya vista esté desarrollada con ExtJS. * Diferencia explícita entre desarrolladores de aplicaciones como el usuario tipo de gvNIX, y usuarios finales de la aplicaciones desarrolladas. * No interfiere en el despliegue de las soluciones ni crea dependencias con el producto final resulta transparente para el despliegue en producción al no contener ningún componente en tiempo de ejecución. --- - Genera cualquiera de las capas de la arquitectura de la aplicación. - Diferencia clara entre desarrolladores y usuarios finales. - Transparente para el despliegue en producción.
# Proyecto de código abierto Herramienta software libre y de código abierto entra las que destaca su núcleo, Spring Roo. Esto le proporciona un amplio apoyo por parte de importantes organizaciones como Spring Source y VMWare. ## Beneficios * La evolucion de las aplicaciones no está bloqueda por la evolución de gvNIX. Integrar una nueva funcionalidad en una aplicación no requiere que previamente se añada a gvNIX. * Corregir incidencias en la aplicación no depende de gvNIX, se pueden corregir en la propia aplicación y posteriormente se reporta al proyecto gvNIX para incluir la mejora en futuras versiones.
